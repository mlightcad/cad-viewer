# 修复 @mlightcad/cad-viewer-example 构建失败问题

## 问题描述

构建 `@mlightcad/cad-viewer-example` 时出现 TypeScript 错误：

```
src/App.vue(28,29): error TS7016: Could not find a declaration file for module '@mlightcad/cad-viewer'.
src/main.ts(4,22): error TS7016: Could not find a declaration file for module '@mlightcad/cad-viewer'.
```

## 根本原因

`@mlightcad/cad-viewer` 包的构建过程中没有正确生成 TypeScript 类型声明文件（`.d.ts`），导致依赖它的 example 项目无法找到类型定义。

## 修复内容

### 1. `packages/cad-viewer/tsconfig.json`

添加类型声明相关配置：

```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./dist",
    "baseUrl": "./src",
    "declaration": true,           // 新增：生成声明文件
    "declarationMap": true,        // 新增：生成声明映射
    "emitDeclarationOnly": true,   // 新增：仅输出声明文件
    "lib": ["ESNext", "DOM"]
  },
  "vueCompilerOptions": {
    "target": 3
  },
  "include": ["src/**/*.ts", "src/**/*.d.ts", "src/**/*.tsx", "src/**/*.vue"]
}
```

### 2. `packages/cad-viewer/package.json`

修改构建脚本和类型入口：

```json
{
  "scripts": {
    "build": "vue-tsc --declaration --emitDeclarationOnly --outDir lib && vite build"
  },
  "types": "./lib/index.d.ts",
  "exports": {
    "types": "./lib/index.d.ts",
    "import": "./dist/index.js"
  }
}
```

### 3. `packages/cad-viewer/vite.config.ts`

为 `vite-plugin-dts` 添加输出目录配置：

```typescript
dts({
  include: ['src/**/*.ts', 'src/**/*.vue'],
  exclude: ['src/**/*.spec.ts', 'src/**/*.test.ts'],
  outDir: 'dist'  // 新增
}) as PluginOption
```

## 验证

执行以下命令验证修复：

```bash
# 清除缓存并重新构建 cad-viewer
npx nx run @mlightcad/cad-viewer:build --skip-nx-cache

# 构建 example 项目
npx nx run @mlightcad/cad-viewer-example:build --skip-nx-cache
```

构建成功，类型文件已正确生成到 `lib/` 目录。
